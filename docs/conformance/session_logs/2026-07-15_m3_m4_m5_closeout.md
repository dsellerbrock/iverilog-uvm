## M5 — interfaces/modports: G26-G29 CLOSED (c399625)

Module ports of interface type were nonfunctional (typedef fallback
degraded them to 32-bit wires). Now, consistent with this fork's
interfaces-as-classes model:
- interface type names resolve to the interface class (elab_type);
- interface ports are class-handle VARIABLES (elab_sig) bound at
  instantiation by an init-scheduled `formal = <vif of actual>`
  (elaborate_mod_), with `<inst>` and `<inst>.<modport>` actuals
  resolved directly to the instance scope;
- a port actual naming an existing instance no longer spawns a
  phantom implicit wire (PExpr.cc);
- nodangle preserves interface-scope members (handles resolve them
  by name at runtime);
- G26: modport import/export task/function ports accepted + recorded
  (PModport::import_ports/export_ports).

Verified working: plain interface ports end to end; bus_if.mst
formals; b.mst actuals (G29); implicit modport (G27); interface
tasks through modport ports; instance arrays + vif arrays (G28).
