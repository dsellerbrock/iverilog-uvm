# 2026-07-15 — M9 increment 1: core SVA engine (PR #76)

Directive: "Implement a proper version of M9." Gaps G05/G06. Before
this increment, `assert property` was a parse-time approximation:
expressions evaluated live in an always block, `|=>` silently treated
as `|->` (an off-by-one-cycle miscompile), no sequence operators,
sampled-value functions as constant-returning VPI stubs, and
property/sequence declarations swallowed by parser error recovery.

## Architecture: synthesized token-pipeline checkers

A concurrent assertion lowers at parse time (pform_make_assertion,
pform.cc) to a clocked checker built on the standard hardware-checker
construction:

- A sequence is a linear chain of cycle-delayed booleans
  (`sva_seq_step_t`): `e0 ##d1 e1 ##d2 e2 ... [##[m:n] eK]` — fixed
  constant delays with at most one trailing bounded range.
- When the sampled antecedent holds, an attempt TOKEN is injected
  into a bank of 1-bit pipeline registers and shifts one position per
  clocking event. Each fixed step checks tokens of exactly its age
  (pre-shift positions); a failing token is killed and the fail flag
  raised. Overlapping attempts are simply multiple tokens in flight —
  no sharing, no threads, fully deterministic.
- A trailing `##[m:n]` is a window register bank: a token completing
  the fixed prefix enters at age 0; the awaited boolean satisfies
  EVERY attempt whose eligible window (positions m..n) covers the
  current cycle; a token shifting past position n unsatisfied fails.
- `|=>` adds one leading cycle to the consequent chain — real
  next-cycle semantics.
- `disable iff` (own, or the module's `default disable iff`, cloned
  per assertion) clears all in-flight state instead of evaluating.
- Assertions with no clocking event use the module's default clocking
  (16.14.6) via the $ivl_default_clock marker from the M8 tail work.
- assume == assert (simulation semantics); cover property/sequence
  count matches into a synthesized 32-bit counter; restrict property
  is ignored per 16.8 (formal-only directive).

Sampling model: every step boolean (and the antecedent) is captured
into a 1-bit sample register at the top of the checker, at the
clocking event in the Active region — before that edge's NBA updates
land, i.e. equal to the Preponed (sampled) value for NBA-driven
logic. Blocking-assignment races against the clock edge fall outside
this equivalence (they are races under event semantics too;
recorded).

## Sampled-value functions (16.9.3)

$rose/$fell/$stable/$changed/$past[,N] inside assertion expressions
get true clocked-history semantics: a clone-on-change PExpr rewriter
(subclass-preserving for PEBComp/PEBLogic/PEBShift) replaces each
call with references to a synthesized current-sample register and an
N-deep history chain, captured at the top of the checker and shifted
at the bottom. Histories are zero-initialized for deterministic
first-cycle behavior (the strict LRM x-default would make $stable
false on cycle one; recorded deviation, matches common tool
practice). Procedural (non-assertion) uses keep the legacy VPI stubs
(recorded).

## Named declarations

No-argument `property p; <spec>; endproperty` and `sequence s;
<chain>; endsequence` (with optional end labels) now parse for real
and register in module-scoped tables. `assert property (p);`
instantiates the named property; a chain step naming a sequence
splices its steps (delays compose). Bodies clone structurally for
reuse; unclonable bodies are consume-once with a clear diagnostic on
the second use. Parameterized declarations still fall to the
pre-existing error-recovery rules (parsed and dropped — recorded).

## Honest sorries (never silent)

Sequence antecedents (`s1 |-> ...` with delays on the left), ranges
anywhere but the final step, unbounded `##[m:$]`, non-constant delay
expressions, and chains spanning more than 512 cycles are all
diagnosed and the assertion dropped. Repetition operators
(`[*N]`, `[->]`, `[=]`), `throughout`, `intersect`, `first_match`,
and property operators (`not`, `until`, `nexttime`) remain syntax
errors as before — the recorded M9-2 frontier.

## Tests

tests/m9_sva_engine_test.sv — 24 checks across two modules, each
supported form exercised in BOTH passing and failing directions:
same-cycle |->; real |=> (no same-cycle check, passes on next-cycle
consequent, failure lands one cycle later); ##2 exact-cycle pass and
fail; ##[1:3] window satisfied mid-window and expiring; disable iff
killing a pending obligation; overlapping back-to-back attempts
tracked separately; $rose/$fell firing exactly on edges; $stable
false only on change cycles; $past reading the previous cycle; a
named property invoking a named sequence (pass + fail); cover
counting; and a second module driven entirely by default clocking +
default disable iff. Legacy sva_default/sva_sequence/assert_property
tests pass unchanged.
