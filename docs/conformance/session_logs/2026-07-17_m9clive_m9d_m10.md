# 2026-07-17 — M9C-live, M9D, M10 (liveness, parameterized SVA, wide DPI)

Directive: "do M9C-live, M9D, M10." Three milestones, each delivered as a
validated increment with its own commit and regression gate.

## M9C-live — SVA liveness operators (IEEE 1800-2017 16.12.2/16.12.5)

`nexttime` / `s_nexttime` / `s_eventually`, boolean operands, reusing the
per-cycle-check + `pend`/FINAL machinery from the `until` family.

- `nexttime p` (weak): aggregated over per-clock attempts, fail at cycle
  T≥1 when `!p`; a `$past(1,1)` guard suppresses the first cycle.
- `s_nexttime p` (strong): same, plus the final attempt has no next
  cycle → one end-of-simulation strong failure.
- `s_eventually p`: p must hold at least once → end-of-sim failure if it
  never did.

Unbounded `eventually` is now a proper LRM error (it requires a cycle
range; use `s_eventually`). The `s_nexttime` grammar rule was added.
op_types 9/10/11 dispatched through `pform_make_temporal_assertion_`.
Test `m9c_live_test`; negatives cover the unbounded-`eventually` error
and the sequence-operand sorry. Commit `39e4af7`.

## M9D — parameterized properties and sequences (IEEE 1800-2017 16.8/16.12)

Named property/sequence declarations may now take formal arguments,
substituted with the actual argument expressions at each instantiation.
Previously these declarations were parsed-and-dropped by bison error
recovery (a reference silently failed).

- Grammar: `sequence name(formals); …; endsequence` and `property
  name(formals); …; endproperty`, plus a formal-name list. Zero new
  bison conflicts.
- `sva_clone_subst_` clones an expression tree replacing formal
  identifiers with clones of the actual arguments (also handles
  `$rose`/`$past`/… calls containing a formal).
- Sequence instantiation `name(args)` expands in the splice pass (arity
  check + recursion-depth guard; nested parameterized sequences
  supported). Property instantiation `p(args)` expands in
  `pform_make_assertion`, taking the clock from the assertion site; a
  clock embedded in the parameterized body is a loud sorry (would need
  per-instantiation event cloning).
- Scope: signal/boolean formals only — a formal used as a `##` delay
  bound is already a non-constant delay, diagnosed at lowering.

Test `m9d_param_test`: `sab(x,y)` vs `sab(y,x)` produce different match
sets, proving per-instantiation substitution; `pab(x,y)` proves the
implication. Negatives: arity mismatch, clocked-body sorry. Commit
`dedd110`.

The remaining M9D items (local sequence variables, `.matched`, `expect`,
goto/nonconsecutive repetition `[->n]`/`[=n]`) genuinely need an
automaton-based sequence engine, not the current flat linear chain, and
remain unimplemented (loud syntax/sorry today).

## M10 — DPI packed vector marshaling (IEEE 1800-2017 35.5.6)

Removes the 64-bit limit on by-value DPI import arguments. A 2-state
`bit [W-1:0]` argument marshals as `svBitVecVal[]` (letter 'V'); a
4-state `logic [W-1:0]` as `svLogicVecVal[]` (letter 'W', aval/bval
pairs, X/Z preserved). Both are passed as a pointer to a packed buffer —
input, output, and inout, any width.

- tgt-vvp (`vvp_process.c`): the >64-bit LOGIC/BOOL argument emits 'W'/'V'
  instead of the old sorry; the output width-atom restriction no longer
  applies to wide vectors. The existing `%load/vec4` seed and
  `%store/vec4` copy-back move the vec4 both ways.
- vvp runtime (`vthread.cc`, `vvp_dpi.cc`): `dpi_call_common_` packs the
  popped vec4 into a heap buffer, passes the pointer via libffi
  (`ffi_type_pointer`, like an open-array handle — no by-value
  indirection or scalar readback), unpacks the callee-updated buffer
  back into a vec4 for copy-back, and frees it. The non-libffi fallback
  path passes the pointer too. `vvp_dpi_arg_t` gained `vbuf`/`vwid`.

Test `m10_dpi_wide_vector_test` (72-bit / 3-word): 2-state xor, 4-state
copy preserving X/Z, and a 4-state read+write invert (complement known
bits, keep X/Z). Existing narrow DPI tests unaffected. Commit `5f09e4d`.

M10B remaining: multi-dimensional open arrays and DPI **export**
(C-calls-SV, the largest remaining DPI item).
